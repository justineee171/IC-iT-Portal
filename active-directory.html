Set Up Active Directory Domain Services (AD DS)

What is Active Directory?

Active Directory (AD) is a directory service developed by Microsoft for Windows domain networks. It stores information about objects on the network and makes this information easy for administrators and users to find and use. AD DS (Domain Services) allows centralized management of users, computers, and other resources.

Step 1: Install the AD DS Role

1. Open Server Manager.
2. Click on Manage > Add Roles and Features.
3. Use the wizard to select Role-based or feature-based installation.
4. Choose your local server and then select Active Directory Domain Services.
5. Click Next and then Install.

Tip: Do not close the wizard until the installation is complete.

Step 2: Promote Server to a Domain Controller

After the role is installed, a yellow notification flag will appear at the top of Server Manager. Click it and choose Promote this server to a domain controller.

  • To start a new network, choose Add a new forest.
  • Provide a root domain name (e.g., mycompany.local).
  • Set a secure Directory Services Restore Mode (DSRM) password.

Step 3: Review & Complete Setup

The wizard checks for prerequisites and displays a summary of configurations. Click Install to begin promotion. Your server will reboot after setup is complete.

Note: You must be logged in as a local administrator to promote the server.

Step 4: Verify Domain Controller Status

  • Open Active Directory Users and Computers to manage objects.
  • Check DNS Manager for correct domain zone creation.
  • Use dcdiag in Command Prompt to check domain health:
dcdiag /v

Step 5: Organize Your Directory

Use Organizational Units (OUs) to keep users and computers organized.

  • Create OUs by department or function (e.g., HR, IT, Sales).
  • Move objects into their appropriate OU for easier management.
  • Use Group Policy Objects (GPOs) to enforce security settings.

Step 6: Best Practices for AD Management

  • Keep at least two domain controllers for redundancy.
  • Regularly back up the system state for recovery.
  • Limit admin access and enforce password policies.
  • Monitor logs in Event Viewer for unusual activity.
