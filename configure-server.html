Configure Server Roles and Features

Select Server Roles

Using Server Manager, go to "Manage" > "Add Roles and Features". Select the server you want to configure and choose the roles it will perform, such as DNS, DHCP, or Web Server (IIS). Consider the purpose of your server and the network services it should provide.

Install Active Directory Domain Services (AD DS)

This role allows the server to act as a domain controller. It enables centralized management of users, computers, and network resources. After installation, promote the server to a domain controller and configure the domain name, forest, and replication settings.

Set Up DHCP Server

The Dynamic Host Configuration Protocol (DHCP) server role automatically assigns IP addresses to client devices on the network. Configure IP scopes, exclusions, and lease durations to optimize your IP address distribution.

Install Web Server (IIS)

To host websites or internal web applications, install the Internet Information Services (IIS) role. Configure bindings (HTTP/HTTPS), set up web applications, and apply security policies like request filtering and SSL certificates.

Configure File and Storage Services

Enable this role to manage shared folders, set file permissions, configure disk volumes, and provide network-based storage. It's essential for servers acting as file servers for local or remote users.

Enable Print and Document Services

This role lets you share printers across the network, manage print queues, and deploy printers via Group Policy. Useful in office environments where centralized printer management is required.

Review and Confirm Installation

Before proceeding, review the summary of selected roles and features. Check for any required restarts or dependencies. Click "Install" and monitor the progress. Some roles may prompt additional configuration after the server restarts.

Post-Installation Tasks

After roles are installed:

  • Use Server Manager or PowerShell to complete setup wizards.
  • Verify services are running using services.msc.
  • Apply firewall rules and security hardening measures.
  • Backup role configurations for disaster recovery planning.
