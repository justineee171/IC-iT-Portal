IP Addressing and Subnetting

What is IP Addressing?

An IP (Internet Protocol) address is a unique identifier assigned to each device connected to a network. It enables communication by identifying the source and destination of data packets.

IP addresses are of two types:

  • IPv4: A 32-bit address written as four decimal numbers separated by dots (e.g., 192.168.1.1).
  • IPv6: A 128-bit address written in hexadecimal, separated by colons (e.g., 2001:0db8:85a3::8a2e:0370:7334).

Understanding IPv4 Address Classes

IPv4 addresses are categorized into classes based on their starting bits and range:

  • Class A: 1.0.0.0 to 126.255.255.255 (Large networks)
  • Class B: 128.0.0.0 to 191.255.255.255 (Medium networks)
  • Class C: 192.0.0.0 to 223.255.255.255 (Small networks)
  • Class D: 224.0.0.0 to 239.255.255.255 (Multicasting)
  • Class E: 240.0.0.0 to 255.255.255.255 (Experimental)

What is Subnetting?

Subnetting is the process of dividing a large network into smaller, more manageable sub-networks or subnets. It helps improve performance, security, and efficient IP address allocation.

Subnetting uses a subnet mask to determine which part of the IP address refers to the network and which part refers to the host.

Subnet Masks and CIDR Notation

Common subnet masks and their CIDR (Classless Inter-Domain Routing) equivalents include:

  • /8 = 255.0.0.0 (Class A)
  • /16 = 255.255.0.0 (Class B)
  • /24 = 255.255.255.0 (Class C)

CIDR notation allows flexible division of IP space and is widely used in modern networking.

Calculating Subnets

To calculate the number of subnets and hosts per subnet:

  • Number of subnets = 2n (where n = number of borrowed bits)
  • Hosts per subnet = 2h - 2 (h = number of host bits, subtracting network and broadcast addresses)

Example: For a /26 subnet (255.255.255.192), there are 64 IP addresses, but only 62 usable hosts.

Benefits of Subnetting

  • Improved network performance through reduced broadcast traffic
  • Enhanced security by isolating segments
  • Efficient utilization of IP address space
  • Scalability for growing networks
