Create and Manage Users

Objective

Learn how to create, configure, and manage user accounts in a networked environment. Managing users ensures secure access, maintains proper permissions, and supports organizational policies.

Step 1: Access User Management Console

  • Windows: Open Computer Management → Local Users and Groups.
  • Linux: Use terminal commands like adduser or usermod.

Step 2: Create New User Accounts

  • Enter a unique username.
  • Set an initial password and enforce password policies (length, complexity).
  • Define the home directory and shell (for Linux).
  • Set account expiration or disable after inactivity (optional).

Step 3: Assign User Roles and Groups

  • Group membership defines what resources users can access.
  • Windows: Add users to groups like Administrators, Users, or custom roles.
  • Linux: Use usermod -aG groupname username to assign groups.

Step 4: Assign Permissions and Policies

Permissions and policies control what users can access and how they can interact with resources. Assigning them correctly is essential for both security and functionality.

File and Folder Permissions

  • Windows: Use NTFS permissions in folder properties (Right-click → Properties → Security tab).
  • Linux: Use commands like chmod, chown, and setfacl to set read, write, and execute rights.
  • Apply the Principle of Least Privilege — give users only the access they need.
  • Test user access by logging in or simulating access using admin tools.

Group Policies (Windows Domain Environments)

  • Group Policies (GPOs) are rules applied to users or computers in Active Directory environments.
  • Used to enforce security settings, software restrictions, desktop environments, and more.
  • Edit policies using the Group Policy Management Console (GPMC).
  • Common GPO examples:
    • Force password complexity and expiration
    • Restrict Control Panel access
    • Map network drives or printers automatically
    • Deploy software installations

Policy Best Practices

  • Use naming conventions and document each policy’s purpose.
  • Use security groups to apply policies efficiently across users.
  • Test policies in a staging environment before live deployment.
  • Regularly review and audit existing permissions and GPOs.

Step 5: Test and Verify

  • Log in as the new user or simulate their access with admin tools.
  • Check if permissions are working as intended.
  • Adjust group membership or access settings if issues are found.

Best Practices

  • Use strong password policies and enable password expiration.
  • Review user accounts regularly and remove unused ones.
  • Enable logging and auditing of login events and access activity.
  • Avoid using shared user accounts — each user should have their own login.
  • Document user roles and access permissions for reference and compliance.
