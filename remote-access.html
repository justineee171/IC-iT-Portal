Enable Remote Desktop Services

What is Remote Desktop Services (RDS)?

Remote Desktop Services allow users to remotely access a Windows server or workstation from another location using the Remote Desktop Protocol (RDP). It’s commonly used in enterprise environments for remote administration, virtual desktops, and centralized app hosting.

Enable Remote Desktop on a Server

  • Open Server Manager.
  • Click Local Server in the left pane.
  • Find Remote Desktop and click the status (usually “Disabled”).
  • Select Allow remote connections to this computer.
  • Optionally, check Allow connections only from computers running Remote Desktop with Network Level Authentication for better security.
  • Click OK.

Configure Firewall Rules

  • Go to Control Panel > System and Security > Windows Defender Firewall.
  • Click Allow an app or feature through Windows Defender Firewall.
  • Ensure Remote Desktop is allowed on Domain, Private, and Public networks if needed.

Add Users to Remote Desktop Group

  • Right-click This PC > Properties.
  • Click Remote Settings > Select Users.
  • Click Add and enter the usernames of those who should have RDP access.

Note: Administrators already have access by default.

Install Remote Desktop Services Role (Optional)

  • In Server Manager, go to Manage > Add Roles and Features.
  • Choose Remote Desktop Services installation.
  • Select deployment type (Quick Start or Standard).
  • Choose role services like RD Session Host, RD Licensing, or RD Connection Broker depending on needs.

This is used for centralized remote session management or virtual desktop infrastructure (VDI).

Connect Using Remote Desktop Client

  • On the client computer, open Remote Desktop Connection (search mstsc).
  • Enter the server's IP address or hostname.
  • Click Connect, then log in with a valid user account.

Ensure the server is powered on and reachable over the network.

Best Practices

  • Use strong passwords and consider multi-factor authentication (MFA).
  • Keep Windows and Remote Desktop Services updated.
  • Enable auditing to monitor RDP login attempts.
  • Limit RDP access via firewall or VPN to trusted IP ranges.
  • Consider using RDS Gateway for added security over the internet.
