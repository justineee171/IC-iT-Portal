Inspect and Test Configured Networks

Verify IP Configuration

Use ipconfig (Windows) or ifconfig/ip a (Linux) to inspect the IP settings. Check if the IP address, subnet mask, default gateway, and DNS servers are correctly configured.

Ping Test

Use ping to test reachability:

  • ping 127.0.0.1 – Loopback (to verify TCP/IP stack)
  • ping default gateway – Check router connection
  • ping 8.8.8.8 – Check external internet access

Check Network Devices

Use arp -a to see a list of devices the PC has communicated with. Use network scanners (e.g., Angry IP Scanner) to discover devices on the network.

DNS Resolution Test

Use nslookup or dig to check if domain names are resolving to IPs correctly. This helps isolate DNS issues from general connectivity problems.

Trace Route and Path Testing

Use tracert (Windows) or traceroute (Linux/Mac) to view the path packets take to a remote server. Identify where delays or packet loss occur.

Loopback Test

Ping 127.0.0.1 to check if your network adapter and TCP/IP protocol are functioning properly. If this fails, there is a software issue on your machine.

Firewall & Antivirus Interference

Sometimes, overprotective firewall or antivirus settings block valid network traffic. Temporarily disable them to test connectivity issues — but always re-enable after testing.

Physical Connection Check

Ensure all cables are connected firmly. For Wi-Fi, verify signal strength and that the device is on the correct network. Check router and switch LEDs to confirm proper activity.

Internet Speed Test

Use tools like Speedtest.net to check current upload/download speeds and confirm ISP performance matches expectations.

Basic Troubleshooting Workflow

  • 1️⃣ Check physical connections
  • 2️⃣ Use ipconfig/ping to test local and internet reachability
  • 3️⃣ Test DNS with nslookup
  • 4️⃣ Use tracert or pathping to isolate path issues
  • 5️⃣ Disable firewall temporarily if nothing else works

Follow this order to quickly isolate whether a problem is with your device, the network, DNS, or the internet connection itself.
